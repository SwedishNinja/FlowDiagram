FlowDiagram — DSL Cheatsheet

Everything you can write between @startuml and @enduml v1.10.0

Document skeleton

@startuml
' single-line comment
/' block
   comment '/

… components, connections, flows, stages …
@enduml

Every diagram is wrapped in @startuml / @enduml. Blank lines are fine anywhere.

Components (boxes)

component "API Gateway" as gw
component "Auth" as auth #10b981
component "User DB" as db #blue <<database>>
[Cache] as cache          ' bracket shorthand
  • alias (after as) is what connections refer to
  • #color — hex (no quotes) or a named color, tints the box
  • <<stereotype>> — small category label inside the box

Connections (lines)

gw -> auth as auth_conn : authenticate
auth ..> db                 ' dotted
gw <-> cache                ' bidirectional
auth --> log                ' long arrow
ArrowMeaning
->solid, forward arrowhead
-->solid, long-form arrow
..>dotted line
<->bidirectional arrowheads

as name names the connection so flows can ride it; : label draws a text label. Unnamed connections get auto-ids.

Packages (groups)

package "Backend" as backend #475569 {
  open: true          ' start expanded
  component "Auth" as auth
  package "Storage" as storage {
    component "DB" as db
  }
  package shared       ' reference: adopt a
}                        '   package declared elsewhere
  • Nest packages to any depth; they auto-resize to fit their contents
  • Packages are layers: closed (single box) until clicked open. open: true starts a package expanded
  • package alias (no quotes, no braces) pulls another package into this one

Flows (animated dots) — all properties

@flow login on auth_conn
  data: "JWT token"       speed: 150              direction: forward
  every: 500ms            start_delay: 200ms      color: #FF0088
  after: other_flow  arrival_effect: ripple          trail: true
PropertyExampleWhat it does
data:"JWT token"Text label carried by the leading dot
every:500ms · 2s · 1mSpawn interval — makes the flow continuous
freq:10/s · 30/mSame as every:, expressed as a rate
speed:150Constant dot speed in px/s — travel time = line length ÷ speed. Default: 150
traverse_time:1sLegacy alternative: fixed time per line regardless of length (overrides speed)
start_delay:200msWait before first spawn; for dependent flows, per-arrival latency
direction:forward · reversereverse travels target → source on the same line (request/response)
color:#FF0088 · redDot color; omit to auto-assign from the palette
after:flow_a, flow_bDependent flow: fires once per upstream arrival (all listed must arrive)
arrival_effect:sparksArrival effect override — see the effects card
trail:true · falseComet trail on/off for this flow
Three spawn modes: a rate (every:/freq:) → continuous  ·  after: → event-driven, one dot per upstream arrival  ·  neither → one-shot: exactly one dot (per stage run, or once per replay).

Stages (scenario phases)

@stage login
  @flow post_login on auth_conn
  @flow token on result_conn
    direction: reverse
    after: post_login
@end_stage

@stage sync
  after: login         ' waits for login
  repeat: true         ' loops forever
  color: #3b82f6       ' colors every flow in the stage
  @flow fetch on db_conn
@end_stage
  • Lifecycle: idle → running → completed. No after: → starts immediately
  • Completes when every flow inside arrived at least once this run
  • after: a, b — starts when all listed stages freshly complete
  • repeat: true — restarts after completing
  • color: #hex — unifies the color of all flows in the stage; a flow's own color: overrides it
  • Flows without a rate fire once per stage run

Diagram settings & positions

' top-level defaults for every flow:
arrival_effect: ripple
trail: true

@positions          ' pin layout coordinates
  gw: 0, 0
  auth: 220, -40
  • arrival_effect: — default effect: dissolve (the default) · outline · ripple · fill · sparks · none
  • trail: — comet trails on by default (false if omitted)
  • @positions — written automatically when you drag boxes; negative coords allowed

Arrival effects — what happens when a dot hits a box

dissolve (default)

Ink-drop absorption: a colored plume diffuses into the box from the entry direction and fades out, like dye in water.

outline

The box border lights up from the hit point, spreading both ways with a soft outer glow, then fades.

ripple

Sonar rings expand from the hit point across the box and fade out.

fill

Liquid pours in from the entry side behind an animated wavy surface, fills ~40% of the box, then evaporates.

sparks

The dot shatters into eight sparks that scatter into the box on curved paths and burn out.

none

No effect — and no arrival delay: the arrival counts immediately. Use for instant chaining.

Timing: with any effect (except none) an arrival only counts — for after: dependencies and stage completion — once the effect finishes (~1 s). Downstream flows and the next stage wait for it.
Handoff re-condensation: when the journey continues from that box (a dependent flow or the next stage departs from the same node), every effect plays a two-act version: it runs ~halfway, then reverses toward the exact point where the outgoing line departs — the plume re-condenses, the outline segment slides around the border, ripples contract, the liquid drains out through the exit, sparks swarm back together — morphing into the next flow's color and landing exactly as the next dot spawns. One continuous substance.

Comet trail

trail: true            ' diagram-wide default

@flow stream on pipe
  trail: false        ' per-flow override

The line glows behind the moving dot like a cooling comet tail; after the dot arrives the wake fades out in place. Composes with any arrival effect and never delays arrivals or stages.

Quick recipe — relay with effects

@startuml
arrival_effect: ripple
trail: true

component "Web" as web
component "API" as api
component "DB"  as db
web -> api as req
api -> db  as query

@stage request
  repeat: true
  @flow call on req
    color: blue
  @flow lookup on query
    color: red
    arrival_effect: sparks
    after: call
@end_stage
@enduml

Blue dot → ripple re-condenses at the API exit while morphing to red → red dot sparks into the DB. Repeats forever.
